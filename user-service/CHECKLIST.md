## 🔐 Безопасность

- [x] Пароли хешируются через BCrypt (10 раундов)
- [x] JWT токен содержит роли пользователя
- [x] JWT подписан секретным ключом (HMAC SHA256)
- [x] JWT содержит сроки действия (exp = 24 часа)
- [x] CSRF отключена (для REST API)
- [x] Все запросы требуют аутентификации (кроме /auth/**)
- [x] Доступ контролируется по ролям (@PreAuthorize)

## 📝 API Endpoints

- [x] `POST /auth/register` - регистрация пользователя
  - Параметры: login, password, email
  - Возврат: User объект
  
- [x] `POST /auth/login` - вход и получение JWT
  - Параметры: login, password
  - Возврат: JWT токен (с ролями в payload)
  
- [x] `GET /users` - список всех пользователей
  - Требует: Роль ADMIN
  - Возврат: Список пользователей
  - Доступ: 403 Forbidden если нет ADMIN

## 🎯 Архитектурные решения

- [x] Роли в БД (не enum) - гибкая система
- [x] JWT вместо session cookies - для микросервисов
- [x] Flyway для управления миграциями - версионирование
- [x] Spring Security для RBAC - стандартная практика
- [x] Índексы на frequently queried колонках - производительность
- [x] Many-to-many связь user-roles - один пользователь может иметь множество ролей
